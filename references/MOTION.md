# MOTION.md：动效工艺参考

> 可选模块 `motion`。只在动效承担反馈、空间关系或任务连续性时加载。所有实现继续使用本地 token，遵守 `DESIGN.md` 的招牌曲线、具名时长、NEVERS 和语义 token 规则。

## 1. 先决定该不该动

每个动效先回答两个问题：它解释了什么变化？没有它，用户会误解或错过什么？答不出来就保持静态。

按发生频率决定克制程度：

| 频率 | 决策 |
|---|---|
| 每次点击或输入都会发生 | 极短、低位移，优先即时反馈 |
| 每个任务发生一两次 | 可表达层级、展开来源或完成关系 |
| 偶尔发生的重要转场 | 可有清楚编排，但不能拖慢下一步 |
| 纯装饰且反复出现 | 删除，除非它承担明确品牌识别且不干扰任务 |

高频动效越应该安静。不要让常用控件每次都表演。

## 2. 起点、终点与空间来源

- 动效从触发它的对象或其空间位置开始，避免内容无来源地淡入。
- 展开应保留触发点与内容的关系；收起应回到可理解的归宿。
- 列表插入、删除和排序要让用户看懂“哪一项变了”，不要让整页一起跳。
- 页面转场先保证阅读焦点和任务连续，再考虑戏剧性。
- 尺度变化要有稳定锚点。默认从中心缩放常会破坏空间解释，应按真实来源设置 `transform-origin`。

## 3. 可中断与输入响应

- 用户反向操作、快速重复点击或按 Esc 时，动效必须能中断并收敛到最新状态。
- 不排队播放已经过期的 hover、展开或路由转场。
- 动画期间不要锁住与结果无关的输入；必须锁定时说明原因。
- 拖拽和手势跟手反馈优先，松手后的收敛才使用缓动。
- 完成反馈不能挡住下一个主动作，也不能让用户等待装饰结束。

## 4. 属性与本地 token

- 优先动画 `transform` 和 `opacity`。必要的 `filter` 要控制面积与成本。
- 不动画 `width`、`height`、`top`、`left`、margin 等布局属性；展开内容用可测量结构与合适的合成方案。
- 禁用 `transition-all`。明确列出属性，并使用本地 token：

```css
transition:
  transform var(--duration-quick) var(--ease-signature),
  opacity var(--duration-instant) var(--ease-signature);
```

- 保持现有招牌曲线 `cubic-bezier(0.22, 1, 0.36, 1)`。
- 保持现有时长：`instant 100ms`、`quick 180ms`、`smooth 280ms`。只有真实任务需要不同意图时才新增语义 token，不在组件里硬编码散落数字。

## 5. Reduced Motion

`prefers-reduced-motion: reduce` 不是简单把所有时长设成零：

- 移除大位移、缩放、视差、自动滚动和持续循环。
- 保留必要的状态变化，可改为即时切换或轻微 opacity 反馈。
- 不因减弱动效而隐藏内容、延迟结果或改变任务顺序。
- 视频和自动播放内容提供暂停方式，并避免闪烁与快速重复变化。

## 6. Before / After / Why 评审

重要动效在交付前记录：

```text
Before：变化前用户看到什么、焦点在哪里
After：变化后状态、焦点和下一步是什么
Why：动效解释了哪种因果、空间关系或反馈；为何这个频率和时长合适
```

如果 `Why` 只能写“更高级”或“更生动”，删除该动效。

## 7. 审核

- [ ] 每个动效都有信息或交互目的，并按发生频率控制强度
- [ ] 起点、终点和 `transform-origin` 符合真实空间关系
- [ ] 快速反向操作可中断，不积压过期动画
- [ ] 只动画合成友好属性，使用本地具名时长和招牌曲线
- [ ] reduced motion 下任务、内容和反馈仍完整
- [ ] 重要动效通过 Before / After / Why 评审
