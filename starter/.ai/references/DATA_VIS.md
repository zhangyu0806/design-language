# DATA_VIS.md：数据可视化参考

> 可选模块 `data-vis`。用于图表、指标、趋势和比较。先保证结论真实可读，再套用当前 preset 与语义 token；装饰不能改变数据含义。

## 1. 先写问题，再选图

先用一句话写出用户要回答的问题，并标明比较对象、时间范围、单位和所需动作。没有问题就不要先画图。

| 问题 | 优先表达 |
|---|---|
| 一个关键值是多少 | 带单位、时间和基准的数字，不必强行画图 |
| 类别之间谁高谁低 | 排序条形图或点图 |
| 随时间如何变化 | 折线图；离散周期可用柱形图 |
| 部分与整体如何组成 | 堆叠条形图；类别少且差异明显时才考虑饼图 |
| 两个变量是否相关 | 散点图，并说明样本量与异常值 |
| 分布是否集中或偏斜 | 直方图、箱线图或密度图 |
| 流程在哪里流失 | 漏斗或分阶段表格，保留基数与口径 |

表格适合查精确值，图适合看形状和关系。用户同时需要两者时，让图给结论，表给核对。

## 2. 真实比例与尺度

- 柱形图原则上从零开始；截断轴必须有明确断裂标识和理由。
- 折线图可用非零基线，但要显示刻度，避免夸大微小变化。
- 不用 3D、透视、面积错觉或不等间距坐标制造差异。
- 时间轴保持真实间隔，汇总周期和时区写清楚。
- 双轴图容易制造虚假相关，默认拆图；确需使用时明确单位和原因。
- 百分比必须说明分母，累计值与单期值不能混在同一视觉语法里。

## 3. 缺失、不确定与口径

- 缺失值不是零。用断线、空位、纹理或明确标注区分“没有数据”和“数值为零”。
- 预测、估算和实测使用不同线型或区域，并写清边界。
- 数据更新时间、筛选条件、样本范围和单位放在用户能发现的位置。
- 异常值不静默删除。若视图做截尾或聚合，要说明规则并提供核对入口。
- 精确数字必须有来源；示例数据必须标为示例。

## 4. 颜色与可访问性

- 颜色来自当前 preset 的语义 token，不硬编码原始色，不使用默认 Tailwind 色板。
- 类别色保持稳定，同一对象跨图不要换色。
- 顺序数据用单向明度阶梯，发散数据以有意义的中点为中心。
- 红绿不能作为唯一差异。搭配位置、线型、形状、纹理或文字标签。
- 文本、网格线、标记和 focus 状态满足对比度；交互图表支持键盘探索和可访问名称。
- 提供文字摘要或数据表，让不依赖视觉的人也能获得核心结论。

## 5. 状态与交互

- Loading 保留图表骨架和范围感，不展示会被误认为真实值的随机形状。
- Empty 区分无数据、筛选后为空和尚未接入数据，并给下一步。
- Error 说明失败范围、数据是否可能过期，以及重试或替代查看方式。
- Partial 明确哪些系列或时间段缺失，不把不完整结果包装成完整结论。
- Hover 不能承载唯一信息；触摸、键盘和静态截图也应读到关键值。
- 筛选、缩放和图例开关要显示当前条件，并提供清除或恢复默认方式。

## 6. 交付审核

- [ ] 图表选择直接回答了已写明的问题
- [ ] 轴、基线、间隔、单位、分母和聚合口径真实明确
- [ ] 零值、缺失、估算、预测和异常值没有混淆
- [ ] 颜色来自语义 token，且不作为唯一编码
- [ ] Loading / Empty / Error / Partial 状态完整
- [ ] 键盘、触摸、读屏和静态截图都能获得核心结论
